### Planner Response

Yes, there are several ways to style scrollbars to make them look more premium and integrated with your dark theme. Since we are already using **Radix UI** and **Tailwind CSS** in this project, here are the three best options:

### 1. The Radix UI `ScrollArea` (Recommended)
This is the "premium" approach and is already used for the queries list in your dialogs. It provides:
- **Consistent look** across all browsers (Chrome, Firefox, Safari).
- **Auto-hiding** behavior (only visible when hovering/scrolling).
- **Custom styling** via CSS variables for the thumb and track.
- **Non-invasive layout**: It doesn't push content to the side when it appears.

### 2. Modern CSS Native Styling
We can use standard CSS properties in [globals.css](cci:7://file:///d:/Websites/GMaps_scraper_gosom/leads-command-center/src/app/globals.css:0:0-0:0) for a "thin" and discreet look:
```css
.custom-scrollbar {
  scrollbar-width: thin; /* Firefox */
  scrollbar-color: theme('colors.slate.700') transparent; /* Firefox */
}

/* Chrome, Edge, Safari */
.custom-scrollbar::-webkit-scrollbar {
  width: 6px;
}
.custom-scrollbar::-webkit-scrollbar-thumb {
  background-color: theme('colors.slate.700');
  border-radius: 10px;
}
```

### 3. Tailwind "Glass" Style
We can make the scrollbar thumb more subtle, perhaps using a translucent blue or slate that matches the "App" aesthetic:
- **Thumb**: Slate-800 or Blue-500 with low opacity.
- **Track**: Completely transparent.

**Would you like me to swap the native scroll in the Job Configuration dialog for a Radix `ScrollArea`, or would you prefer a custom CSS style applied globally?** (I'll wait for your go-ahead to implement).
